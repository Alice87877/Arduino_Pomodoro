其中碰到的一些問題

delay()這個函示不適合拿來跑倒數
因為其他函式也需要時間 可能很微小但實際來看時間非常不准
delay()同時會延後其他程式無法同步運行
後用millis()實際運行時間來解決

tinkercad只要程式一多跑起來很卡頓
實際運行時間為現實的2-3倍

tinkercad在倒數時不知為何會突然LCD滅掉 找不到原因
改用arduino ide 時先用Serial Monitor測試(尚未接感測器 單純上傳到板子上測試)卻沒問題
卻認為tinkercad上的問題
